# 项目审查与清理说明

## 审查范围与结论

本次只审查本地 `grok2api-v3-migration` 项目，没有连接或修改远程服务器和
SUB2 实例。

已复查账号选择、失败归类、凭据续期、可选账号清理、注册、Browser Worker、
部署打包和凭据存储路径。当前源码通过下方全部质量门禁，审查范围内没有已知的
阻塞性问题。

## 当前工作区已包含的修复

- 调度会排除禁用、`reauthRequired`、模型不兼容、推理权限拒绝、冷却、额度耗尽
  和并发已满的账号，并返回可区分的失败原因。持久化推理健康状态和有限重试可
  避免持续命中不健康账号。
- 凭据续期、额度恢复、注册结果对账和可选的老旧重授权账号清理都有工作量上限
  和跨实例协调。破坏性清理默认关闭，并跳过存在请求租约或排队/执行中媒体任务
  的账号。
- 协议注册响应缺少 SSO 时不再触发未定义变量错误。Browser Worker 会把错误的
  超时和代理参数作为客户端错误返回，不再变成内部失败。
- OAuth 和账号凭据采用原子替换及受限文件权限写入；日志不再记录 Token、OTP、
  Session 或 OAuth 查询参数。
- Cookie 快照、本地 Token 池、代理认证扩展和 CPA 失败记录在创建时即使用私有权限，
  并通过刷新、同步和原子替换避免半写文件。浏览器诊断 URL、协议 debug URL 和代理
  错误不再暴露 userinfo、query 或 fragment；egress 辅助脚本只输出白名单状态字段。
- Protobuf 配额时间戳和产品码增加了溢出/范围校验；推理恢复路径会检查随机 opaque
  token 生成错误，并关闭所有丢弃或替换的 HTTP 响应体。
- 注册凭据输出已从 Git 和 Docker 构建上下文排除。部署打包脚本会拒绝磁盘根目
  录、仓库本身和仓库上级目录等危险输出位置。
- 本地运行 PID 文件已加入忽略规则，开发实例运行时不会因此污染 Git 状态。

| --- | ---: | --- | --- |

| `deploy_artifact/app` | 60 MiB | 生成的应用副本 | 可由源码和打包脚本重建。 |
| `deploy_artifact/app-release` | 58 MiB | 生成的发布副本 | 可由源码和打包脚本重建。 |

Go 构建和测试会重新编译。完整 Staticcheck 仍会报告既有的 `ST1005` 文案风格和
`U1000`，后者包含供 Swagger 扫描使用的声明及尚未启用的协议转换分支，不应在本次
稳定性修复中批量删除；如要缩减源码，应单独建立删除清单并逐项证明没有生成器或
兼容入口依赖。

## 禁止当作构建产物清理的数据

| 路径 | 当前规模 | 处理要求 |

| 活动的 `.runtime-*` 文件、本地可执行文件和日志 | 当前运行状态 | 正常停止本地服务并确认 PID 已失效前必须保留。 |

`.tmp/ssh-askpass.cmd` 应按凭据辅助文件处理，不应进入发布归档。确认当前流程不再
使用后再删除，并轮换可能在其中保存或经它传递的凭据。任何曾在密钥存储之外公开
发送的 API Key 或服务器凭据也应轮换。

## 建议清理顺序

1. 将 `data/registration`、OAuth 输出、`.env` 和 `config.yaml` 备份到受访问控制
   的位置，并验证备份可读取。
2. 通过正常启动方式停止 `grok2api-local`，确认 PID 对应进程不再存在。
3. 删除不再需要的凭据辅助文件，并轮换可能已暴露的凭据。
4. 选择性清理 `.tmp` 中的过期内容，不要把数据库快照或截图带进部署包。
5. 按需删除 `build`、`dist`、`deploy_artifact/app`、
   `deploy_artifact/app-release` 和 `frontend/dist`。
6. 按需删除编译器、测试、包管理器和依赖缓存。
7. 生成下一版发布包前重新执行验证命令。
